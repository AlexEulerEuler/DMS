# Data Contract

The prototype uses a small normalized document contract before chunking.

## Normalized Document

```json
{
  "doc_id": "stable hash",
  "category": "medication",
  "source_path": "data/raw/medication (1).json",
  "source_type": "json",
  "locator": "record[12]",
  "event_time": "optional ISO-like date string",
  "title": "short source-aware title",
  "text": "retrieval text",
  "metadata": {
    "category": "medication",
    "source_path": "...",
    "source_name": "...",
    "locator": "record[12]"
  }
}
```

## Categories

- `treatment`: outpatient treatment and pharmacy visits
- `checkup`: health checkup records
- `medication`: medication history
- `immunization`: immunization records
- `genomic_vcf`: compressed VCF source
- `genomic_index`: VCF tabix index companion
- `microbiome`: gut microbiome data
- `glucose_report`: continuous glucose report PDF
- `holter_report`: Holter report PDF
- `unknown`: uncategorized source

## Chunk Metadata

Each chunk keeps:

- `chunk_id`
- `doc_id`
- `category`
- `source_path`
- `source_name`
- `locator`
- `chunk_index`
- `text`

This is enough to render a source list even when the final answer is generated by
Upstage.

## API-Backed Sources

For HUB API records, `source_path` is a pseudo URI:

```text
hub:///v1/records/treatments?patientSeq=5030
```

Additional metadata:

- `source_type`: `hub_api`
- `api_endpoint`
- `api_summary`
- `patient_seq`
- `transaction_id`
- `record_seq`
