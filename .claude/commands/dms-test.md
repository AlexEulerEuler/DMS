대상($ARGUMENTS)에 대해 **tester 서브에이전트**(.claude/agents/tester.md)를 스폰해 기획 적합성을 검증하라.
단위 테스트가 아니라 "실제로 기획대로 작동하는가"가 목적이다.

tester에게 전달할 것:
- 검증 기준의 정본: 대상 이슈의 완료 조건 + 관련 스펙 문서(apps/dms-console/docs/spec/ 등)
- 실행 방법: 콘솔 프론트는 `npm run dev:dms:frontend` 후 프리뷰 도구로 확인
- 발견 처리 규칙 (docs/policy/00-doc-system.md §6):
  - **PR에 연결된 검증이면**: 이슈를 만들지 말고 PR 코멘트로 보고 + 사람에게 요약
  - **머지된 코드의 검증이면**: 동일 증상의 open 이슈를 먼저 검색(중복 방지)하고, 없으면
    `type:bug` + `by:agent` 라벨로 이슈 생성 — 문서 근거(경로#절)와 재현 절차 필수.
    한 세션에서 신규 이슈는 최대 5개.

완료 후 검증 항목·통과/실패·발행한 이슈를 사람에게 요약 보고하라.
