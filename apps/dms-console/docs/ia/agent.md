# Agent

## 1. 개요

Agent는 프로젝트 계획 단계에서 "앞으로 만들 에이전트"의 기획서를 텍스트로 적어 두는 도구다. 지금 단계의 목적은 실행이 아니라 기획이다 — 무엇을 하는 에이전트인지, 어떤 입력·동작·도구가 필요한지 아이디어와 설계 메모를 남긴다.

이 모듈은 DMS 콘솔의 런타임 기능이 아니다. 여기서 정의한 에이전트가 콘솔에서 곧바로 실행되지는 않으며, 실제 구현·실행은 별도(향후 LangChain·LangGraph + 툴 조합)로 진행한다. 현재 범위는 목록과 상세(간단 텍스트 기획), 그리고 필요 시 RAG 등 참고 자료 첨부 수준이다.

> 향후 고도화: flow(처리 흐름) 편집기, 플레이그라운드(대화 테스트), 도구·RAG 조합, 실행/배포 연동. 현 문서에서는 자리만 표시하고 상세는 다루지 않는다.

## 2. 내비게이션 위치

1차 메뉴 6개(Overview, Task, WBS, Issues, Work, Agent) 중 여섯 번째다. Agent 선택 시 에이전트 목록(`/agents`)이 기본으로 열린다. 좌측 사이드바 2차 메뉴로 "에이전트 목록"이 노출된다. 등록(new)·상세(:agentId)는 목록에서 진입하는 파생 화면이다.

## 3. 하위 구조 / 라우트

```text
Agent (/agents)
└── 에이전트 목록          /agents
    ├── 에이전트 등록      /agents/new
    └── 에이전트 상세      /agents/:agentId   (기획 텍스트 편집)
```

- `/agents` — 목록. 카드로 조회하고 등록으로 진입.
- `/agents/new` — 등록. 이름·설명·기획 메모 작성.
- `/agents/:agentId` — 상세. 하나의 에이전트 기획을 열람·편집.

`:agentId`가 없는 값이면 "없음" 화면을 표시한다(foundation 공통 상태). 현 범위에서는 상세 하위 탭(flow·플레이그라운드)을 두지 않는다 — 향후 도입 시 4depth 탭으로 확장한다.

## 5. 컴포넌트 인벤토리

목록 뷰

- 페이지 헤더 — 제목(Agent), 설명, 요약(총 개수).
- 에이전트 등록 버튼.
- 에이전트 카드 — 상태 배지, 생성일, 이름, 설명 요약.
- 카드 액션 — 열기, 삭제.
- 빈 상태 안내.

상세 뷰

- 이름·상태(드롭다운)·설명 입력.
- 기획 메모 — 자유 텍스트 본문.
- (선택) 참고 자료 첨부 — RAG 소스·링크.
- 하단 액션 — 삭제, 목록으로, 저장.

## 6. 데이터 모델

현재 범위의 에이전트 엔티티는 기획 텍스트 중심으로 얕다. 공통 정의는 [data-model.md](./data-model.md)를 따른다.

| 필드 | 설명 |
| --- | --- |
| `id` | 에이전트 식별자(라우트 `:agentId`) |
| `name` | 에이전트명 |
| `status` | 기획 상태(7절) |
| `description` | 한 줄 설명 |
| `planNote` | 기획 메모(자유 텍스트) |
| `references` | (선택) 참고 자료·RAG 소스 링크 |
| `createdAt` / `updatedAt` | 생성·수정 시각 |

- `systemPrompt`·`flowDefinition`·`toolRules` 같은 실행용 구조는 현 범위에 없다. 향후 고도화(LangChain·LangGraph) 단계에서 도입한다.
- Work의 작업이 `linkedAgent`로 이 기획을 참조할 수 있다(개발 작업이 구현할 에이전트 기획). 실행 주체가 아니라 참조 관계다.

## 7. 상태 및 전이

기획 진행 상태는 세 단계다. 정본은 [status-taxonomy.md](./status-taxonomy.md)를 따른다.

```text
기획중 → 확정
  │
  └──→ 보류 ──→ (기획중 재개)
```

- 기획중 — 기획 메모를 작성·수정하는 단계. 등록 직후 기본값.
- 확정 — 기획이 정리된 상태(종료). 이후 구현은 별도 진행.
- 보류 — 잠시 멈춘 상태. 재개 시 기획중으로 복귀.

> 실행·검증 관련 상태(테스트·배포 등)는 향후 런타임 도입 시 추가한다.

## 8. 액션·인터랙션

- 에이전트 등록 — 등록 버튼 → `/agents/new` → 이름 등 입력 후 저장.
- 열기 — 카드 → `/agents/:agentId`.
- 편집·저장 — 상세에서 이름·설명·기획 메모 수정 후 저장(`updatedAt` 갱신).
- 삭제 — 목록 카드 또는 상세에서 제거(확인 후).
- 상태 변경 — 상세의 상태 드롭다운.

## 9. 사용자 흐름

```text
Agent 목록
   → 에이전트 등록 (/agents/new)
   → 상세: 이름·설명·기획 메모(목적·입력·동작·도구/RAG) 작성
   → 저장, 필요 시 상태를 확정으로 변경
```

기획이 확정된 에이전트는 이후 Work의 개발 작업(`linkedAgent`)에서 참조해 구현 대상으로 삼는다.

## 10. 엣지 케이스

- 에이전트 0건 — 빈 상태 안내와 등록 유도.
- 필수값 누락 — 이름 없이 저장 시 막고 입력 요구.
- 존재하지 않는 `:agentId` — "없음" 화면.
- 미저장 이동 — 저장하지 않은 변경이 있으면 확인.
- 삭제 후 참조 — Work가 `linkedAgent`로 참조 중이면 끊어진 참조로 처리.

## 12. 오픈 이슈

- 향후 고도화: flow 편집기·플레이그라운드·도구/RAG 조합·실행·배포 연동을 LangChain·LangGraph 기반으로 도입. 진입 시 상세 탭(4depth)과 실행용 필드(systemPrompt 등)·런타임 상태를 추가한다.
- `references`(RAG 소스) 첨부의 구체 형식(파일·URL·컬렉션)은 고도화 시 확정.
- 작업↔에이전트는 단수 연결(N:1). 한 이슈가 여러 에이전트와 관련되면 여러 작업으로 표현. — 결정 완료(5.2).
