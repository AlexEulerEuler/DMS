# Overview

DMS Console의 프로젝트 문서 허브. 파이프라인의 입력·출력과 12개 세부 문서 페이지를 한 곳에서 열람한다.

## 1. 개요

- 목적: 콘솔이 다루는 파이프라인을 사람이 읽을 수 있는 문서로 설명한다. 무엇을 입력받아 무엇을 만들어내는지, 어떤 모듈·명령·환경으로 구성되는지를 한 자리에서 파악하게 한다.
- 범위: 읽기 전용 참조 문서 12종과, 파이프라인 입력/출력을 요약한 카드. 파이프라인 실행(생성)은 CLI/백엔드가 담당하며, Overview는 산출물 조회·다운로드("주요 데이터")까지만 제공하고 데이터 편집은 하지 않는다.
- 주 사용자: 신규 합류자, 운영·기획 담당자, 그리고 구조를 빠르게 확인하려는 개발자.

Overview는 콘솔의 랜딩 지점이자 다른 모듈로 진입하기 전 맥락을 잡는 관문 역할을 한다.

## 2. 내비게이션 위치

- 상위 메뉴: 1차 메뉴 6개(Overview, Task, WBS, Issues, Work, Agent) 중 첫 번째.
- depth: 1depth = Overview, 2depth = 좌측 사이드바의 "프로젝트 문서" 하위 12개 문서.
- 진입: 상단 로고 또는 `Overview` 탭 클릭. 앱 루트(`/`)는 `/overview`로 이어진다.
- 이탈: 상단 1차 메뉴로 다른 모듈 이동, 또는 좌측에서 다른 문서 선택.
- 기본 라우트: `/overview` (= "프로젝트 개요" 문서).

## 3. 하위 구조 / 라우트 맵

```text
Overview  (/overview)
└── 프로젝트 문서
    ├── 프로젝트 개요      /overview
    ├── 도메인 용어        /overview/glossary
    ├── 전체 파이프라인    /overview/pipeline
    ├── 서비스 모듈        /overview/modules
    ├── API 엔드포인트     /overview/api
    ├── CLI 명령어         /overview/cli
    ├── 폴더 구조          /overview/folder-structure
    ├── 주요 데이터        /overview/data
    ├── 기술 스택          /overview/tech-stack
    ├── 환경 변수          /overview/env
    ├── 실행 방법          /overview/runbook
    └── 알려진 한계        /overview/limitations
```

12개 문서는 모두 같은 레이아웃을 공유하는 형제 페이지다. 사이드바 그룹 라벨 "프로젝트 문서" 자체는 링크가 아니라 섹션 헤더다.

각 문서의 목적:

| 문서 | 라우트 | 목적 |
| --- | --- | --- |
| 프로젝트 개요 | /overview | 입력·출력·워크플로우 요약(기본 화면) |
| 도메인 용어 | /overview/glossary | 콘솔 핵심 용어 정의(글로서리) |
| 전체 파이프라인 | /overview/pipeline | 입력→처리→출력 단계 상세 |
| 서비스 모듈 | /overview/modules | 구성 모듈과 책임 범위 |
| API 엔드포인트 | /overview/api | 제공 API 목록·용도 |
| CLI 명령어 | /overview/cli | 생성 실행 등 명령어 레퍼런스 |
| 폴더 구조 | /overview/folder-structure | 코드·산출물 디렉터리 구조 |
| 주요 데이터 | /overview/data | 생성 산출물 조회·다운로드(표준 목록·생성 일정·내보내기 파일) |
| 기술 스택 | /overview/tech-stack | 사용 기술·의존성 |
| 환경 변수 | /overview/env | 구동에 필요한 환경 변수 |
| 실행 방법 | /overview/runbook | 로컬·운영 실행 절차 |
| 알려진 한계 | /overview/limitations | 제약·미지원 사항 |

### 3.1 용어(글로서리)

`/overview/glossary`가 담는 콘솔 핵심 용어. 엔티티 상세 정의는 [data-model.md](./data-model.md)를 따른다.

| 용어 | 뜻 |
| --- | --- |
| 표준 목록(Master List) | 입력 문서·기존 표준 목록을 근거로 생성·확정하는 통합 목록 |
| 입력 문서(Source Document) | 생성 기준이 되는 원본 입력 |
| 기존 표준 목록(Existing Master List) | 생성 시 참조하는 이미 확보된 표준 목록 |
| 기준 일정(Baseline Schedule) | 생성 일정 산출의 기준선 |
| 생성 일정(Generated Schedule) | 산출된 문서별 제출 일정 |
| 내보내기 파일(Export File) | JSON/XLSX/문서로 내보낸 결과 |
| 마일스톤 | 제출 기준 시점(유형 A/B) |
| 진행률 | 상태 기반 평균(정의는 [wbs.md](./wbs.md) 7.3) |

## 4. 화면 레이아웃

```text
┌───────────────────────────────────────────────────────────┐
│ 상단 바: 서비스명 · 1차 메뉴 6 · (우) 설정                  │
├───────────┬───────────────────────────────────────────────┤
│ 좌측       │ 콘텐츠 영역                                    │
│ 사이드바   │  제목 + 설명                                   │
│ 프로젝트   │  메타 칩(버전·언어·브랜치·패키지·API)          │
│ 문서 12    │  개요 문단                                     │
│ (선택=파랑)│  입력/출력 카드 그리드                         │
│            │  핵심 워크플로우(단계 흐름도)                  │
└───────────┴───────────────────────────────────────────────┘
```

- 상단 바, 좌측 사이드바는 콘솔 공통 셸(참조: [foundation.md](./foundation.md)).
- 콘텐츠 영역은 선택된 문서에 따라 바뀐다. 아래 컴포넌트 목록은 기본 문서인 "프로젝트 개요" 기준.

## 5. 컴포넌트 인벤토리

프로젝트 개요 화면 기준.

- 페이지 헤더: 제목("프로젝트 개요"), 한 줄 설명("입력 문서 기반 표준 목록 자동 생성 시스템").
- 메타 칩 행: 버전, 언어, 브랜치, 패키지, API. 라벨+값 형태의 읽기 전용 배지.
- 개요 문단: 파이프라인이 무엇을 받아 무엇을 만드는지 서술.
- 입력 카드(3): 각 카드 = 제목 + 설명.
  - 입력 1 — 입력 문서(Source Document)
  - 입력 2 — 기존 표준 목록(Existing Master List)
  - 입력 3 — 기준 일정(Baseline Schedule)
- 출력 카드(1): 생성 일정(Generated Schedule), 표준 목록, 내보내기 파일(JSON/XLSX/문서)을 요약.
- 핵심 워크플로우: 단계 흐름을 좌→우/상→하로 표현한 다이어그램(데이터 기반 렌더).
- 좌측 사이드바 문서 메뉴 12개(3절 참조).

## 6. 데이터 모델

Overview는 참조 콘텐츠 중심이라 자체 엔터티는 얕다. 표시 값(프로젝트 메타·파이프라인 요약)은 원격 API로 불러온다(런타임 조회).

```text
ProjectMeta
├── version         버전
├── language        언어/런타임
├── branch          브랜치
├── package         패키지/빌드
└── apiEndpoint     API 진입점

PipelineSummary
├── inputs[]        { key: input1|input2|input3, title, description }
├── outputs[]       { title, description }   // 생성 일정 · 표준 목록 · 내보내기 파일
└── workflowSteps[] { order, label }         // 업로드 → … → 일정 생성
```

- 입력 유형은 입력 문서 / 기존 표준 목록 / 기준 일정 세 가지로 고정.
- 상세 스키마는 [data-model.md](./data-model.md) 참조.

## 7. 상태 및 전이

문서 콘텐츠와 메타를 원격 API로 불러오므로 로딩/에러 상태가 실제로 쓰인다.

- empty: 특정 문서에 표시할 내용이 아직 없을 때. "문서 준비 중" 안내와 다른 문서로의 링크.
- loading: 메타·요약 로드 중. 헤더/카드 자리에 스켈레톤.
- error: 로드 실패. 오류 메시지 + 다시 시도. 사이드바 내비게이션은 계속 동작.
- populated: 헤더·메타·카드·워크플로우가 모두 렌더된 정상 상태.

전이: loading → populated | error, error →(재시도) loading, populated →(문서 전환) loading.

## 8. 주요 액션 · 인터랙션

- 문서 전환: 좌측 사이드바에서 12개 문서 간 이동. 선택 항목은 파란 배경으로 강조.
- 링크 이동: 워크플로우/카드 안의 참조에서 관련 모듈로 이동(예: 일정 생성 → WBS).
- 메타 칩: 읽기 전용. 클릭 동작 없음(값 표시 용도).
- 설정: 상단 공통 도구(콘솔 전역).

Overview는 조회 중심이라 생성·편집·삭제 액션이 없다. 다만 "주요 데이터"(/overview/data)에서 생성 산출물(표준 목록·생성 일정·내보내기 파일)의 조회와 파일 다운로드는 예외적으로 제공한다. 파이프라인 실행(생성)은 현 범위에서 CLI/백엔드가 담당하고("CLI 명령어" 문서 참조), 편집·상태 변경은 Task/Work/Agent가 담당한다.

## 9. 관련 사용자 흐름

파이프라인 이해 흐름:

```text
Overview 프로젝트 개요
→ 입력 카드로 입력 3종 확인
→ 핵심 워크플로우로 처리 단계 파악
→ 전체 파이프라인 / 서비스 모듈 문서로 상세 확인
→ 실제 관리 화면(Task·WBS·Work)으로 이동
```

신규 합류자 온보딩 흐름:

```text
Overview 프로젝트 개요
→ 도메인 용어로 기본 개념 습득
→ 기술 스택 / 폴더 구조 / 실행 방법 확인
→ 환경 변수 · API · CLI로 로컬 구동
```

## 10. 엣지 케이스 / 예외

- 잘못된 문서 슬러그(`/overview/unknown`): "없음" 화면을 표시한다(foundation 공통 상태).
- 메타 일부 누락: 값 없는 칩은 숨기고 나머지는 정상 표시.
- 입력/출력 요약 미제공: 해당 카드 영역을 비우고 안내 문구로 대체.
- 긴 문서 본문: 콘텐츠 영역 세로 스크롤. 사이드바·상단 바는 고정.
- 좁은 화면: 사이드바 접힘/토글, 카드 그리드는 1열로 재배치.
